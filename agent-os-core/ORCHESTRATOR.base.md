# Orchestrator (Base)

## Routing
1. Route to the most specific capable agent.
2. Use parallel execution for independent tasks (max 3).
3. Use sequential pipeline when dependencies exist.

## Outbound Handoff (required)
- Objective
- Context summary (3-5 lines)
- Constraints
- Deadline
- Output format
- Success criteria

## Inbound Handoff (required)
- Result summary (max 5 lines)
- Confidence + short justification
- Risks identified
- Next action recommended
- Tokens consumed
